# TASK_BOARD

## 当前阶段
`H5 原型收口 + 微信小程序实现对齐 + MVP 主链路 + 社区 UI 前端 + Shop 导购壳`

## 当前阶段目标
把《拍立食》从静态 H5 原型推进到可分工落地的微信小程序项目：
- 主链路可演示。
- Nexus 成为首页中枢。
- 社区以前端展示形态进入首页和社区页。
- Shop 作为从共享菜谱跳到第三方买菜平台的导购壳成立。
- 前端、后端、测试、临时文件、开发文档目录分层清楚。

## 当前进行中
- 统一微信小程序最终技术口径，不再混用 React/Vite 说法。
- 把当前 H5 原型目录整理为正式前端原型资产。
- 让后端/API/数据库/上架同学能基于统一文档分工。
- 继续把主链路和社区导购逻辑从 H5 原型迁移到小程序正式目录。

## 当前已收口规则
- 底部第二栏统一为 `社区`。
- 中间 `+` 统一直达 `拍摄.html`。
- 不再保留文字/语音输入弹层。
- 拍摄页主 CTA 为 `确认拍摄`，随后进入分子重构确认页。
- 分子重构确认页底部 CTA 负责先进入加载过渡页的确认菜单，再从确认菜单进入加载。
- 食材数量优先显示口语化家庭单位，而不是克重。
- 社区承担“看到别人分享 -> 想跟做 -> 去第三方平台买食材”的引导职责。

## 下一个任务
- 把 `frontend/miniprogram/` 搭起来。
- 明确微信登录、上传、识别、生成、归档四条接口链路。
- 整理社区页与 Shop 跳转页的字段结构。
- 确认 Shop 在小程序中的正式跳转方案。
- 为结果页后续的健康参考模块预留展示位置和数据接口。

- 登录 / Nexus / 拍摄识别 / 分子重构确认 / 加载过渡 / 结果页 / 保存归档 / 档案馆 全链路。
- Nexus 首页中枢。

- 最终技术栈口径已经统一为微信小程序。
- Nexus 可用。
- 档案馆可用。
- 社区 UI 可见且逻辑自洽。
- Shop 跳转壳可用。
- 没有因为社区或 Shop 把项目复杂度拉爆。
