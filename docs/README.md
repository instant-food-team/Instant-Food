# 拍立食 Docs Guide

`docs/` 是项目文档入口。协作、交接、答辩准备都以这里为准。

## 建议先读
1. `docs/h5_acceptance_standard.md`
2. `docs/architecture/TECH_ARCHITECTURE.md`
3. `docs/architecture/ARCHITECTURE_RULES.md`
4. `docs/planning/PROJECT_SCOPE.md`
5. `docs/planning/TASK_BOARD.md`
6. `docs/ui/UI_STYLE_BASELINE.md`
7. `docs/handoff/TEAM_TRANSFER_GUIDE.md`

## 目录职责
- `architecture/`：技术架构、边界与协作约束
- `planning/`：范围、里程碑、任务板
- `backend/`：数据模型、接口契约与模板
- `ui/`：UI 视觉基线与实现提示
- `handoff/`：交接、同步与接手说明
- `presentations/`：汇报文档与素材
- `reference/`：课程与外部参考资料

## H5 验收约束（强制）
- 所有 H5 改动必须遵循：`docs/h5_acceptance_standard.md`
- 未通过必过项前，不得切换正式入口
- 验收证据归档路径：`archive/validation/<date>-<topic>`
