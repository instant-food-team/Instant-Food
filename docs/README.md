# 拍立食 Docs Guide

`docs/` 是项目的正式文档入口。组员协作、Google Drive 上传、交接阅读，都优先以这里为准。

## 先读顺序
1. `docs/architecture/TECH_ARCHITECTURE.md`
2. `docs/architecture/ARCHITECTURE_RULES.md`
3. `docs/planning/PROJECT_SCOPE.md`
4. `docs/planning/TASK_BOARD.md`
5. `docs/ui/UI_STYLE_BASELINE.md`
6. `docs/ui/UI_NEW_WINDOW_PROMPT.md`
7. `docs/handoff/HANDOFF.md`
8. `docs/handoff/TEAM_TRANSFER_GUIDE.md`

## 目录职责
- `architecture/`
  - 技术架构、边界、协作约束、团队同步包
- `planning/`
  - 项目范围、阶段目标、任务板
- `backend/`
  - API 契约、数据模型、健康模块数据模板
- `ui/`
  - UI 风格基线、新窗口提示词
- `handoff/`
  - 交接记录、团队同步模板、文件转交清单
- `presentations/`
  - 汇报和 PPT 底稿
- `reference/`
  - 课程资料和外部参考
