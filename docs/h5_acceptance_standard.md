# 拍立食 H5 验收标准

## 目的
这份文档定义 `拍立食` H5 的固定验收标准。
后续所有修改都必须按这套标准验收，包括：
- 壳页重构
- 图片替换
- 文案替换
- 交互修复
- API 接入
- Supabase 接入
- 部署路径调整

任何版本只要有必过项不通过，就不能切成正式入口。

- `C:\Users\Yang\Desktop\拍立食\frontend\vercel.json`

## 验收范围
这套标准覆盖以下页面和组件：
- 引导页
- 登录页
- 首页
- 社区页
- 拍摄页
- 档案馆页
- 阶段 3 页面
- 我页 / 设置页
- 顶部栏
- 底部栏
- 弹层 / 抽屉 / 遮罩
- 页面切换和返回逻辑

## 发布门槛
一个版本只有在以下条件全部满足时，才能切换为正式入口：
- 所有必过项全部通过
- 没有阻断级交互问题
- 没有相对稳定版的可见视觉回归
- 没有丢图、空图、错图
- 没有异常黑遮罩、黑条、暗层
- 主导航和返回链路全部可用

## 问题分级
- `P0`：页面不可用、进不去、退不回、核心图片丢失、严重布局损坏
- `P1`：明显视觉回归、按钮无响应、路由不通、异常遮罩、演示质量受影响
- `P2`：细小视觉不一致、边距轻微异常、非阻断动画差异

发布规则：
- `P0 = 0`
- `P1 = 0`
- `P2` 只有在你明确接受时才允许存在

## 一、视觉一致性
### 必过项
- 页面结构必须与当前稳定版保持一比一
- 不允许出现意外黑遮罩、黑条、暗带、雾层
- 不允许出现图片拉伸、裁切错误、留白不足、图片没铺满容器
- 不允许文字被裁切、重叠、乱码、行高异常
- 同一页面内字体风格必须统一
- 顶栏、底栏、卡片、按钮、标签样式必须与稳定版一致
- 任何卡片、按钮、标签不得错位或被挡住

### 检查点
- 对照稳定版截图检查
- 单独检查顶部区域、主体区域、底部区域
- 重点检查首页、社区、拍摄、档案馆

## 二、资源完整性
### 必过项
- 所有图片必须正常加载
- 不允许出现空白图框、破图、占位块、背景图缺失
- 所有图标必须正常显示
- 不允许因为本地路径、部署路径、嵌套 iframe 路径导致资源失效

### 检查点
- 引导页背景图
- 首页卡片图
- 社区卡片图
- 档案馆卡片图
- 拍摄页图片与底部区域
- 顶部和底部所有图标

## 三、导航与返回
### 必过项
- 所有主导航跳转必须可用
- 所有返回按钮、关闭按钮、左上角 `X` 必须可用
- 不允许出现死路页面
- 返回必须回到预期页面，而不是无反应或跳错页
- 页面切换不能依赖用户手动刷新

### 必测链路
- 引导页 1 -> 引导页 2 -> 引导页 3
- 引导页 -> 登录页
- 登录页 -> 首页
- 首页 -> 社区
- 首页 -> 拍摄
- 首页 -> 档案馆
- 拍摄页 -> 左上角 `X` -> 返回上一页
- 社区 -> 卡片打开逻辑 -> 返回
- 档案馆 -> 返回
- 设置 / 我页 -> 返回

## 四、交互可用性
### 必过项
- 可拖动控件必须跟手
- 滑动和拖动过程中，视觉反馈必须正常
- 可点击按钮必须首击有效
- 不允许出现“看起来能点，但点了没反应”的控件
- 弹层、抽屉、遮罩的开关逻辑必须正常
- 滚动区域必须能自然滚动，不能被壳页错误拦截

### 必测交互
- 引导页 `即刻料理` 拖动必须跟手
- 引导页右滑 / 拖动切换必须正常
- 登录页按钮响应正常
- 拍摄页左上角 `X` 正常关闭 / 返回
- 拍摄页底部确认区域响应正常
- 档案馆筛选标签可切换
- 社区卡片点击逻辑正常
- 底部导航所有按钮都可点

## 五、遮罩与暗层控制
### 必过项
- 页面顶部不能出现无设计依据的黑条
- 页面底部不能出现无设计依据的黑条
- 切页完成后，遮罩必须消失
- 不允许存在透明但拦截点击的残留层
- 登录页、拍摄页、首页、档案馆页都必须重点检查遮罩问题

### 典型失败示例
- 顶部黑带
- 底部栏后面多出一层黑背景
- 页面上方或下方出现暗雾层
- 切场动画结束后遮罩没撤掉
- 按钮点不动，因为上面压了透明层

## 六、布局与安全区
### 必过项
- 内容不能被顶部或底部安全区切掉
- 底部 CTA / 确认按钮必须完整可见
- 底部导航不能意外压住重要内容
- 安全区和边距行为必须与稳定版一致
- 桌面预览和移动端视口都必须正确

### 基线视口
- 以当前原型验证视口为主：`414 x 896`
- 必查顶部间距
- 必查底部栏间距
- 必查拍摄页底部操作区域

## 七、文案与图片替换验收
### 当修改文案或图片时，必须额外检查
- 新文案不能导致换行异常、截断、重叠
- 新图片必须铺满原有容器，不得露底、不足、不变形
- 纯文案或纯图片修改也不能破坏交互和跳转
- 不允许出现“只是换图换文案，但页面逻辑坏了”的情况

## 八、API / Supabase 接入验收
### 当接入真实数据时，必须额外检查
- loading 状态不能破坏布局
- empty state 必须是有设计的空状态，不是空白页
- error state 不能直接把原始报错丢到界面上
- 数据长度变化不能造成卡片溢出、文字压住、图片塌陷
- 异步加载后不能重新引入遮罩、死点击、错位
- 从详情页返回的逻辑必须仍然正常

### 必测数据场景
- 短标题
- 长标题
- 无图片
- 无副标题
- 空列表
- 接口慢返回
- 接口报错

## 九、固定回归清单
每次做完补丁后，必须至少跑完下面这组回归。

### A. 截图回归
必查页面：
- 引导页 1
- 引导页 2
- 引导页 3
- 登录页
- 首页
- 社区页
- 拍摄页
- 档案馆页
- 阶段 3 页面
- 我页 / 设置页

要求：
- 与稳定版无可见回归
- 关键页在可行时做像素对比

### B. 功能回归
必查项：
- 所有左上角返回 / `X`
- 所有右上角应有响应的按钮
- 所有底部导航
- 引导页拖动
- 拍摄页进入与退出
- 档案馆筛选与返回
- 社区打开与返回
- 设置页弹层打开与关闭

### C. 视觉点检
必查项：
- 顶部无黑条
- 底部无黑条
- 图片完整铺满容器
- 字体统一
- 小视口下无裁切
- 无额外阴影遮罩层

### D. 部署路径检查
部署版本必须额外检查：
- HTTPS 部署地址可直接打开
- 嵌套路由下图片路径仍然正确
- 部署后不丢图
- 不依赖 `file://` 本地双击行为

## 十、执行流程
以后每次做中等以上修改，都按这个流程执行：
1. 先识别这次改动影响哪些页面和交互
2. 只修改最小责任层
3. 跑受影响页面的截图回归
4. 跑受影响链路的功能回归
5. 跑遮罩、底栏、顶栏、图片铺满的视觉点检
6. 把证据归档到 `C:\Users\Yang\Desktop\拍立食\archive\validation\<日期>-<主题>`
7. 只有全部通过，才算这次修改完成
