# UI_STYLE_BASELINE

## 1. 用途
这份文档是《拍立食》当前 UI 的强约束基线。后续任何 Codex 在新窗口做 UI、改页面、补页面、补组件时，都必须先读这份文档，再开始动手。

优先级说明：
- 这份文档只约束 UI 风格与交互实现。
- 产品边界以 `docs/planning/PROJECT_SCOPE.md` 为准。
- 架构与禁止事项以 `docs/architecture/ARCHITECTURE_RULES.md` 为准。
- 如果视觉参考和当前页面冲突，以 `frontend/prototype/Chinese/` 现有页面风格为准，不允许借题发挥做新设计系统。

## 2. 当前视觉定义
《拍立食》当前不是通用电商 UI，也不是明亮型菜谱 App。它的视觉语言已经固定为：
- 移动端优先
- 深色高质感
- 苹果系排版
- 液态玻璃 / 黑曜石玻璃
- 轻电影感
- 克制的绿色品牌光

关键词：
- 黑底
- 深灰玻璃面
- 绿色主品牌
- 鼠尾草色辅助高光
- SF Pro 字体
- 大圆角卡片
- 柔和模糊
- 轻微漂浮
- 微弱发光

## 4. 颜色规范

### 固定品牌色
- 主品牌色：`#235347`
- 辅助强调色：`#8EB69B`

### 固定背景色
- 页面底色：`#000000`
- 深面板：`#151515` 到 `#1C1C1E`
- 深层玻璃面：`rgba(22, 22, 22, 0.88)` 到 `rgba(10, 10, 10, 0.96)`

### 固定文字色
- 主文字：`#FFFFFF`
- 次文字：`#8E8E93`
- 弱化文字：`rgba(255, 255, 255, 0.72)` 到 `rgba(255, 255, 255, 0.78)`

### 使用规则
- `#235347` 只能承担主 CTA、主按钮、主状态高光、选中态主色。
- `#8EB69B` 只用于辅助点亮、标签高光、状态提示，不抢主品牌色。
- 不允许把 `#00E5FF`、`#FF9500`、紫色、亮蓝色做成新的主品牌色。

## 5. 字体规范

### 固定字体
- 标题字体：`SF Pro Display`
- 正文字体：`SF Pro Text`

### 中文回退
- `PingFang SC`
- `Microsoft YaHei`
- 系统 sans

### 使用规则
- 页面主标题、区块标题、关键数值使用 `SF Pro Display`。
- 正文、说明、标签、按钮文案使用 `SF Pro Text`。
- 标题字重优先 `600` 到 `700`。
- 正文字重优先 `400` 到 `600`。
- 标题默认使用轻微负字距：`-0.02em` 到 `-0.03em`。

## 6. 间距、圆角、阴影

### 间距体系
固定按 `8 / 12 / 16 / 20 / 24 / 32` 体系执行。

### 圆角体系
- 设备外壳：`40px`
- 主卡片：`28px`
- 次级卡片：`22px` 到 `26px`
- 小控件：`12px` 到 `18px`
- 胶囊按钮 / 标签：`999px`

### 阴影体系
- 深阴影：`0 30px 60px rgba(0, 0, 0, 0.42~0.45)`
- 软阴影：`0 18px 40px rgba(0, 0, 0, 0.28)`
- 内高光：`inset 0 1px 0 rgba(255, 255, 255, 0.04~0.08)`

规则：
- 圆角整体要大，不要突然出现锐角矩形。
- 阴影要软、要深、要像悬浮在黑色背景上。
- 不要使用高对比硬阴影。

## 7. 液态玻璃规则
项目允许使用液态玻璃，但必须是深色、克制、偏黑曜石玻璃，不是亮白 frosted glass。

允许的表现：
- 深色半透明面板
- `backdrop-filter: blur(12px~28px)`
- 极细白色描边
- 顶部轻内高光
- 绿色或鼠尾草色微弱发光

禁止的表现：
- 大面积亮白毛玻璃
- 彩色炫光
- 透明到看不清结构
- 每个组件各搞一套玻璃参数

## 8. 固定交互规则
以下是当前已经锁定的交互，不允许偏离：
- 底部导航统一为 `首页 / 社区 / 中间主入口 / 记录 / 我`。
- 底部第二栏固定为 `社区`。
- 中间 `+` 固定直达 `拍摄.html`。
- 不再弹出文字、语音或工具输入小框。
- 拍摄页主 CTA 固定为 `确认拍摄`，点击后进入 `分子重构台.html`。
- 分子重构确认页中的食材控件、保留/剔除控件、技法/厨具/味觉选择控件必须可点击。
- 分子重构确认页底部主 CTA 先进入 `加载过渡.html` 的确认菜单态，再由确认菜单进入加载生成。
- 首页当前不保留 `最近档案` 和 `身份摘要` 两个模块。
- 分子重构确认页、加载过渡页、结果页兜底文案中的数量展示优先使用 `块 / 片 / 颗 / 瓣` 这类口语单位。

## 9. 组件风格要求

### 按钮
- 主按钮使用主品牌绿色。
- 次按钮使用深色玻璃。
- 按钮高度优先 `56px~60px`。
- 文字清晰，不做过小轻字。

### 卡片
- 主卡片必须是深色玻璃或深色实体。
- 卡片内边距优先 `18px~24px`。
- 卡片必须有明确边界和层次，不要漂成一团。

### 底部导航
- 保持深色玻璃 dock 风格。
- 中间主入口可以更强，但只能有一个视觉中心。
- Dock 外壳统一参数：`left/right 20px`、`bottom 8px`、`padding 10px 8px 8px`、`border-radius 28px`
- 中间主入口统一为 `52px * 52px` 圆形，悬浮偏移 `top: -18px`

## 10. 动效规则
动效必须轻、短、稳。

允许：
- `180ms~240ms` 的 hover / press / 状态切换
- `translateY(-1px~-2px)` 的轻微悬浮
- `scale(0.98~0.99)` 的点击反馈
- 加载阶段的轻呼吸光、微旋转、轻过渡

禁止：
- 大位移动画
- 复杂弹簧动画
- 高频闪烁
- 炫技型页面切场

## 11. 明确禁止项
- 不准改成浅色主界面
- 不准换品牌色
- 不准用随机字体
- 不准做成通用电商、通用社区、通用 SaaS 风格
- 不准把社区或 Shop 做成另一套视觉系统
- 不准为了“更完整”重构无关页面

## 12. 一句话记忆
《拍立食》的 UI 必须始终保持为：

`苹果系 SF Pro 排版 + 黑底深灰玻璃 + #235347 主品牌色 + #8EB69B 辅助高光 + 大圆角 + 深色液态玻璃 + 移动端单屏原型`
